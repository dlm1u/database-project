# Автоматизированная система логистики склада

---

## О проекте
 
Данная база данных предназначена для автоматизации учета товаров на складе, отслеживания их перемещений между зонами и ячейками, а также контроля работы персонала с входящими поставками и исходящими заказами.

Система позволяет:
- контролировать остатки товаров  
- обрабатывать заказы  
- анализировать загрузку сотрудников  
- отслеживать изменения цен товаров

## Структура базы данных
В базе реализовано 6 таблиц:
* `CLIENT` — информация о клиенте.
* `PRODUCT` — перечень товаров.
* `WAREHOUSE_ZONE` — сведения о складах.
* `SUPPLIER` — данные о поставщиках.
* `EMPLOYEE` — Информация о сотрудниках.
* `ORDER` — сведения о заказах.

Таблица `ORDER` является центральной и связывает все остальные сущности.

## Дополнительный функционал

### Представления
#### `order_full_info`
Представление, объединяющее данные из нескольких таблиц:
- клиент
- товар
- склад
- сотрудник

Позволяет:
- упростить сложные JOIN-запросы  
- использовать готовую бизнес-логику  

Тип: **обычное представление**  
Причина: требуется актуальность данных в реальном времени  

#### `employee_workload_mv`
Материализованное представление, отображающее:
- общее количество заказов на сотрудника  
- количество заказов в обработке  

Тип: **materialized view**  
Причина: содержит агрегаты (COUNT), которые дорого пересчитывать  

---
### Триггеры
#### Резервирование товара
Срабатывает при создании заказа
- Проверяет наличие товара на складе
- Уменьшает количество товара

**Назначение:** предотвращает создание заказов без достаточного количества товара

#### Возврат товара при отмене
- Срабатывает при изменении статуса заказ
- Возвращает товар на склад при отмене

Особенность: учитывает изменение статуса, чтобы избежать повторного возврата
---
### Версионность данных (SCD Type 2)

Для таблицы PRODUCT реализовано хранение истории изменения цены.
Добавлены поля:
- valid_from — начало действия версии
- valid_to — конец действия
- is_current — актуальная версия

Особенности: сохраняется полная история изменений, можно анализировать данные во времени
